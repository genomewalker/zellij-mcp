Capabilities

Agent-grade terminal control

⬡

Agent Session Isolation

Autonomous operations run in a separate zellij-agent session. Your workspace stays untouched—no focus stealing, no visual interruption.

◫

Named Pane Targeting

Address any pane by name, not just the focused one. Create "worker", target "worker", read "worker"—across tabs and sessions.

⏱

Output Monitoring

wait_for_output, wait_for_idle, tail_pane. Watch for patterns, detect completion, stream incremental output.

⌘

REPL Interaction

Execute code in IPython, R, Julia. Auto-detect REPL type, send code blocks, capture output with prompt detection.

☁

SSH & HPC Workflows

ssh_connect to remote hosts, ssh_run commands, job_submit to SLURM/PBS clusters, job_status tracking.

⬢

Agent Spawning

spawn_agents to launch multiple Claude instances on parallel tasks. Track status, read output, coordinate work.

Workflow

Agent Isolation

By default, all autonomous operations run in a separate zellij-agent session. Claude creates panes, runs commands, monitors output—completely invisible to your active workspace.

# Your session stays untouched
$ zellij list-sessions
main (current)
zellij-agent
 
# Claude works in isolation
create_named_pane name="worker" command="python3"
→ Created in zellij-agent
Integration

Pairs with prism.nvim

The complete development workflow: zellij-mcp manages panes, prism.nvim handles editor integration. Open nvim in a pane, edit visibly, run tests in another pane.

# Full workflow
1. zellij-mcp: "open nvim in a floating pane"
2. prism.nvim: "go to line 42, add error handling"
3. zellij-mcp: "run tests in a pane below"
4. zellij-mcp: "capture the test output"
